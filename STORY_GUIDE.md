# 故事书添加指南

## 如何添加新的故事书

### 1. 确定故事ID
选择一个新的故事ID，格式为3位数字，如：
- `001` 
- `002`
- `003`
- `011` (当前最新)

### 2. 准备图片文件
在 `public/stories/` 目录下创建对应的故事文件夹，例如 `012/`

需要准备以下图片文件：
- `cover.png` - 故事封面图片
- `01.png` 到 `10.png` - 故事内容页图片（共10张，支持扩展页面如11.png, 12.png等）

图片规范：
- 格式：PNG 推荐，也支持 JPG/JPEG
- 尺寸：建议 1200x800px (3:2 比例)
- 文件大小：尽量控制在 500KB 以内以优化加载速度
- 命名要求：严格遵循两位数字格式（01.png, 02.png...）

### 3. 创建故事配置文件
在 `src/config/` 目录下创建对应的YAML配置文件，例如 `012.yaml`

配置文件示例：
```yaml
title: "故事标题"
author: "若兰"  # 可以是AI助手名、真实作者或"AI小助手"
createdDate: "2025-08-26"
tags: ["冒险", "友情", "动物", "教育"]

description: "这是一个关于友情和勇气的温馨故事"  # 可选：故事简介

pages:
  - "第一页的故事文字内容，通常是对场景的描写..."
  - "第二页的文字，可以是对话或者叙述..."
  - "第三页的文字..."
  - "第四页的文字..."
  - "第五页的文字..."
  - "第六页的文字..."
  - "第七页的文字..."
  - "第八页的文字..."
  - "第九页的文字..."
  - "第十页的文字..."
  # 支持扩展页面，如:
  - "第十一页的文字..."  # 如果页面超过了10页
  - "第十二页的文字..."
```

### 4. 音频文件（可选）
在 `public/audios/` 目录下创建对应的故事文件夹，例如 `012/`

#### 自动音频生成（推荐）
使用内置TTS脚本生成高质量中文音频：
```bash
# 生成指定故事的音频
npm run tts 012

# 生成所有故事的音频
npm run tts:all

# 查看音频生成状态
npm run tts:status
```

需要先配置TTS API密钥（`.env`文件）：
```
MINIMAX_GROUPID=your_group_id
MINIMAX_API_KEY=your_api_key
```

#### 手动添加音频文件
音频文件：
- `01.mp3` 到 `10.mp3` - 对应每页的朗读音频（支持扩展如11.mp3, 12.mp3等）
- 格式：MP3
- 音质：128kbps 或更高
- 时长：建议每段 10-30 秒
- 命名要求：严格遵循两位数字格式（01.mp3, 02.mp3...）

### 5. 文件结构示例（故事012）
```
public/
  stories/
    012/
      cover.png
      01.png
      02.png
      ...
      10.png
      11.png  # 可选扩展页面
      12.png  # 可选扩展页面

  audios/          # 自动生成或手动添加
    012/
      01.mp3
      02.mp3
      ...
      10.mp3
      11.mp3  # 对应扩展页面
      12.mp3  # 对应扩展页面

src/
  config/
    012.yaml       # 故事配置
```

### 6. 构建和测试
1. 添加所有文件后，重启开发服务器：`npm run dev`
2. 访问 `http://localhost:4321` 查看新故事是否出现在首页
3. 点击故事卡片进入阅读页面测试所有功能
4. 测试音频播放（如已生成）
5. 测试不同设备的响应式效果（PC、平板、手机）

### 7. 音频测试
如启用了音频功能：
```bash
# 测试音频生成
npm run tts 012
```
测试音频播放：
- 播放/暂停功能
- 播放进度控制
- 播放速度调节（0.5x-2x）
- 自动翻页和连续播放

### 注意事项 ✅
- ✅ 确保文件名严格按照规范命名（3位数字故事ID）
- ✅ YAML文件中的`pages`数组元素数量必须与图片文件一致（包括扩展页面）
- ✅ 图片文件命名必须是两位数字格式（01.png, 02.png等）
- ✅ 音频文件命名必须与图片对应（01.mp3对应01.png）
- ✅ 如果没有音频文件，音频控制区域会自动隐藏
- ✅ 检查YAML格式是否正确（注意冒号和缩进）

### 8. 一键脚本
使用一键脚本自动完成故事创建：
```bash
# 详细文档见 scripts/README.md
node scripts/import-gemini-story.js --help
```

## 🔧 高级功能

### 批量处理建议
- **图片处理：** 推荐使用ImageMagick或Photoshop批处理统一图片尺寸
- **文件命名：** 使用批处理工具保持命名一致性
- **YAML编辑：** 使用VS Code的多光标功能批量编辑
- **模板使用：** 复制现有配置文件作为模板

### 自动化流程
1. **Gemini导入：** 使用 `scripts/import-gemini-story.js` 从Gemini响应直接生成完整故事包
2. **自动音频：** 使用TTS脚本一次性完成所有音频生成
3. **自动验证：** 文件添加后会自动验证格式和完整性

### 故障排除
- **图片不显示：** 检查文件路径和命名是否正确
- **音频不播放：** 确认浏览器支持MP3格式，检查控制台错误信息
- **YAML错误：** 使用YAML验证工具检查格式
- **页面数量不匹配：** 确保pages数组长度与图片文件数量一致

## 故事阅读功能说明

### 基础功能
- ✅ 翻页导航（上一页/下一页按钮）
- ✅ 键盘控制（左右方向键）
- ✅ 触摸滑动（移动端）
- ✅ 页码显示
- ✅ 全屏阅读模式
- ✅ 响应式布局

### 音频功能 ✅
- ✅ **音频播放/暂停** - 点击播放按钮控制播放状态
- ✅ **播放进度显示** - 可视化进度条，可拖拽定位
- ✅ **播放速度调节** - 支持0.5x至2x倍速播放
- ✅ **自动翻页** - 根据音频时长自动切换到下一页
- ✅ **连续播放** - 故事内页面自动连续播放

**使用条件：** 当对应故事的音频文件存在时自动启用

**文件检测：** 系统会检测 `public/audios/XXX/` 目录下的音频文件，如存在则显示音频控制界面，否则自动隐藏
